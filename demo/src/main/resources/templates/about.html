Sobre o Pinboard

Pinboard

Aplicação desenvolvida com Spring Boot utilizando diversos padrões de projeto.

Padrões de Projeto Implementados

Singleton

O padrão Singleton garante que uma classe tenha apenas uma instância e fornece um ponto global de acesso a ela.

Implementação:

  • ConfigurationManager: Gerencia as configurações globais da aplicação.
  • Permite acesso centralizado a parâmetros de configuração como nome da aplicação, versão e modo de depuração.
  • Disponível em toda a aplicação através do método estático getInstance().

Com este padrão, garantimos que apenas uma instância de configuração seja usada em todo o sistema, mantendo consistência nos parâmetros da aplicação.

Observer

O padrão Observer define uma dependência um-para-muitos entre objetos, de modo que quando um objeto muda de estado, todos os seus dependentes são notificados e atualizados automaticamente.

Componentes implementados:

  • PinSubject (interface): Define os métodos para gerenciar observadores.
  • PinObserver (interface): Define o método de atualização.
  • PinNotificationService: Implementação concreta do Subject que notifica os observadores.
  • ActivityLogObserver: Implementação concreta do Observer que registra atividades.

Este padrão é utilizado para registrar atividades no sistema, como criação de pins, likes e outras interações dos usuários.

Factory

O padrão Factory fornece uma interface para criar objetos em uma superclasse, mas permite que as subclasses alterem o tipo de objetos que serão criados.

Componentes implementados:

  • PinLayout (interface): Define a interface para os layouts de exibição.
  • LayoutFactory: Fábrica que cria diferentes tipos de layouts (Grid, List, Masonry).
  • Implementações concretas: GridLayout, ListLayout, MasonryLayout.

Com este padrão, podemos criar diferentes layouts para exibição de pins sem acoplar o código cliente aos tipos concretos de layout.

Decorator

O padrão Decorator permite adicionar novas funcionalidades a objetos existentes sem alterar sua estrutura, encapsulando o objeto em objetos decoradores.

Componentes implementados:

  • PinComponent (interface): Define a interface comum para componentes e decoradores.
  • BasicPin: Implementação básica de um pin.
  • PinDecorator: Classe abstrata base para decoradores de pins.
  • Decoradores concretos: FeaturedPinDecorator, SocialSharingDecorator.

Este padrão nos permite adicionar dinamicamente recursos a pins, como destaque visual para pins populares ou funcionalidades de compartilhamento.

Strategy

O padrão Strategy define uma família de algoritmos, encapsula cada um deles e os torna intercambiáveis. Permite que o algoritmo varie independentemente dos clientes que o utilizam.

Componentes implementados:

  • SortingStrategy (interface): Define a interface para algoritmos de ordenação.
  • Estratégias concretas: DateStrategy (ordenação por data), PopularityStrategy (ordenação por número de likes).
  • PinService: Utiliza as estratégias para fornecer diferentes métodos de ordenação de pins.

Com este padrão, permitimos aos usuários alternar entre diferentes critérios de ordenação para visualizar pins, como por data de criação ou por popularidade.

Tecnologias de Persistência

Banco de Dados

O Pinboard utiliza PostgreSQL como solução de persistência de dados, um banco de dados relacional robusto e open-source, amplamente utilizado em ambientes de produção.

Características do banco de dados:

  • PostgreSQL: Banco de dados relacional open-source, poderoso e extensível, ideal para aplicações em produção.
  • Persistência Durável: Dados armazenados de forma segura com suporte a transações ACID.
  • JPA/Hibernate: Framework ORM para mapeamento objeto-relacional.
  • Modo Update: Esquema do banco é atualizado automaticamente quando as entidades do modelo são modificadas.
  • Configuração via Variáveis de Ambiente: Conexão configurável através de variáveis DATABASE_URL ou PGHOST, PGPORT, PGDATABASE, PGUSER e PGPASSWORD.

Configurações de conexão:

  • URL: jdbc:postgresql://localhost:5432/pinboard (padrão local)
  • Usuário: postgres
  • Senha: configurada via variável de ambiente PGPASSWORD

Esta implementação garante que todos os dados criados pelos usuários (pins, boards, perfis, etc.) sejam preservados de forma segura e consistente, proporcionando uma experiência confiável em ambiente de produção.
